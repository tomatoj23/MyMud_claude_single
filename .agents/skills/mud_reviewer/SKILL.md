# MUD代码审查助手

## 功能概述

本 Skill 提供《金庸武侠文字MUD》项目的代码审查支持：

1. **审查清单** - 各核心系统的详细检查项
2. **测试模板** - 单元测试和集成测试模板
3. **验收标准** - 各阶段的具体验收检查项
4. **问题分级** - 阻塞/严重/警告/建议的分级标准

## 使用场景

- 审查 mud-coder 生成的代码
- 执行单元测试和集成测试
- 对照验收标准检查阶段完成度
- 生成审查报告和问题清单

## 问题严重程度分级

| 级别 | 定义 | 处理方式 |
|:---|:---|:---|
| **阻塞** | 导致系统无法运行，或严重违反架构设计 | 必须修复，不通过审查 |
| **严重** | 功能缺陷，性能问题，安全隐患 | 必须修复，不通过审查 |
| **警告** | 代码异味，可维护性问题 | 建议修复，可记录待处理 |
| **建议** | 风格优化，文档改进 | 可选，记录即可 |

## 性能指标要求

根据开发计划，系统必须满足以下性能指标：

| 指标 | 目标值 | 检查方式 |
|:---|:---|:---|
| 单场景对象数 | 支持100+ | 压力测试 |
| 命令响应时间 | < 100ms | 性能测试 |
| 内存占用 | < 500MB | 内存监控 |
| 启动时间 | < 5秒 | 计时测试 |
| 存档加载时间 | < 3秒 | 计时测试 |
| 自动存档卡顿 | < 1秒 | 用户体验测试 |

## 测试覆盖要求

- **单元测试覆盖率**：> 80%
- **类型注解覆盖率**：100%
- **核心模块测试**：Typeclass、ObjectManager、Command、EventScheduler必须有完整测试

### 审查清单覆盖范围

**核心系统**：Typeclass、ObjectManager、Command、EventScheduler、GameEngine、存档系统  
**游戏内容**：Character、装备、武学、地图、战斗、任务、NPC  
**GUI系统**：PySide6集成、qasync桥接、面板、主题  
**代码质量**：类型安全、异步规范、错误处理、代码风格

## 审查输出格式

```markdown
## 审查摘要
- 审查模块: xxx
- 代码文件数: n
- 审查结果: [通过/有条件通过/不通过]

## 逐项检查

### 1. 架构符合性
- [x] 类设计符合开发计划
- [x] 接口定义一致
- [x] 继承关系正确

### 2. 代码质量
- [x] 类型注解完整
- [x] 文档字符串规范
- [x] 错误处理完善

### 3. 功能测试
- [x] 单元测试通过
- [x] 集成测试通过

## 问题清单
| 严重程度 | 位置 | 问题描述 | 建议修复 |
|:---|:---|:---|:---|
| 严重 | file.py:42 | xxx | xxx |
| 警告 | file.py:88 | yyy | yyy |

## 结论
[通过/需修复]
```

## 审查工作流程

1. **读取检查清单** - 根据模块类型查阅 checklists.md
2. **执行代码审查** - 逐条检查架构、质量、风格
3. **编写并运行测试** - 使用 tests.md 中的模板
4. **对照验收标准** - 查阅 acceptance.md 确认阶段要求
5. **输出审查报告** - 按格式输出结论

## 示例：审查 Typeclass 实现

```
用户：审查刚实现的 Typeclass 代码

→ 读取 references/checklists.md 中的 Typeclass 检查项
→ 逐项检查代码
→ 读取 references/tests.md 获取测试模板
→ 生成并运行测试
→ 按格式输出审查报告
```
