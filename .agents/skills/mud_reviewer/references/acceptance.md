# 各阶段验收标准

## 阶段一：引擎核心搭建

### 功能验收

- [ ] 可通过命令行与引擎交互
- [ ] 支持创建对象、移动、查看等基本操作
- [ ] 事件调度器可正确处理延迟/周期任务
- [ ] 对象修改后正确标记dirty并持久化

### 技术验收

- [ ] Typeclass系统支持动态类加载
- [ ] AttributeHandler透明代理属性访问
- [ ] ObjectManager实现L1/L2缓存
- [ ] CmdSet支持优先级合并
- [ ] 命令解析使用Trie树前缀匹配
- [ ] EventScheduler使用asyncio实现
- [ ] 所有核心模块有单元测试

## 阶段二：武侠世界构建

### 功能验收

- [ ] 可创建完整武侠角色（含先天/后天属性）
- [ ] 角色可装备/卸下武器、防具
- [ ] 角色可学习武功并使用招式
- [ ] 地图系统支持三维移动和寻路

### 技术验收

- [ ] Character属性三层结构清晰
- [ ] 装备系统支持12+槽位
- [ ] 武学招式脚本沙箱安全
- [ ] A*寻路算法正确实现
- [ ] 属性计算无循环依赖

## 阶段三：玩法系统实现

### 功能验收

- [ ] 完整的战斗流程（开始→行动→结算→结束）
- [ ] 可接受、进行、完成任务
- [ ] NPC有日常行为并与玩家对话
- [ ] 支持自然语言命令

### 技术验收

- [ ] 战斗系统支持回合/即时双模式
- [ ] BUFF/DEBUFF系统正确计算
- [ ] 任务状态机无死锁
- [ ] NPC行为树可配置
- [ ] 自然语言解析准确率达80%+

## 阶段四：GUI客户端开发

### 功能验收

- [ ] 所有游戏功能可通过GUI操作
- [ ] 主题切换即时生效
- [ ] 响应式UI，状态变更实时反映
- [ ] 支持快捷键操作

### 技术验收

- [ ] PySide6与asyncio正确桥接
- [ ] 无界面卡顿或崩溃
- [ ] QSS样式系统可扩展
- [ ] 支持窗口缩放自适应

## 阶段五：内容制作与集成

### 功能验收

- [ ] 可创建角色并选择门派
- [ ] 可完成至少3条主线任务
- [ ] 可学习门派基础武学
- [ ] 可进行NPC对话和战斗

### 数据验收

- [ ] 所有房间有完整描述（100-300字）
- [ ] 十大门派各有基础武学（外功、内功、轻功）
- [ ] 主线任务链完整（4章）
- [ ] 50+有完整对话的NPC
- [ ] 20+可战斗的NPC
- [ ] 10+商人NPC

### 世界数据验收

**房间数据检查：**
- [ ] 每个房间有唯一的key
- [ ] 房间名称不超过20字
- [ ] 房间描述在100-300字之间
- [ ] 出口连接到有效的房间key
- [ ] 坐标设置合理（避免重叠）

**武学数据检查：**
- [ ] 每种武功有唯一的key
- [ ] 招式伤害倍率在合理范围（1.0-3.0）
- [ ] 内力消耗与伤害成比例
- [ ] 冷却时间设置合理
- [ ] 克制关系配置完整

**任务数据检查：**
- [ ] 每个任务有明确的目标描述
- [ ] 前置条件可验证
- [ ] 奖励配置合理
- [ ] 任务链连接正确

**NPC数据检查：**
- [ ] NPC有日常行程安排
- [ ] 对话树无死循环
- [ ] 战斗AI配置合理
- [ ] 好感度变化逻辑正确

## 阶段六：存档与系统功能

### 功能验收

- [ ] 存档/读档功能完整
- [ ] 自动存档正常触发
- [ ] 存档版本兼容性处理正确
- [ ] 开发者模式可实时查看/修改数据
- [ ] 对象浏览器可查看和编辑游戏对象
- [ ] 战斗模拟器可批量测试并生成报告
- [ ] 命令控制台可执行开发者命令

### 技术验收

**存档系统检查：**
- [ ] MessagePack序列化正确
- [ ] 存档压缩后大小合理（<10MB）
- [ ] 存档加密（可选）实现正确
- [ ] 自动存档不卡顿（<1秒）
- [ ] 存档版本迁移逻辑正确

**开发者模式检查：**
- [ ] 对象浏览器显示所有对象
- [ ] 属性修改实时生效
- [ ] 日志查看器支持筛选和搜索
- [ ] 性能监控显示FPS、内存、对象数
- [ ] 平衡测试可运行1000+场战斗
- [ ] 控制台命令执行正确

### 性能指标

- [ ] 单场景支持100+对象
- [ ] 命令响应时间 < 100ms
- [ ] 内存占用 < 500MB（单机版）
- [ ] 启动时间 < 5秒
