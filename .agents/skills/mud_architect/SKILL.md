# MUD项目架构规划助手

## 功能概述

本 Skill 提供《金庸武侠文字MUD》项目的完整开发规划参考，包括：

1. **四阶段开发路线图** - 引擎核心 → 武侠世界 → 玩法系统 → GUI客户端
2. **模块清单与依赖** - 每阶段的详细模块分解和依赖关系
3. **验收标准** - 各阶段的具体验收检查项
4. **激活模板** - 用于协调其他专家的指令模板

## 使用场景

- 启动新开发阶段时查阅阶段目标
- 分解当前阶段的模块任务
- 检查阶段完成度和验收标准
- 生成激活 mud-coder 和 mud-reviewer 的指令

## 开发阶段总览

```
阶段一：引擎核心搭建（第1-3周）
├── 项目骨架、数据库基础设施、ORM模型
├── Typeclass动态类系统、ObjectManager对象管理器
├── CmdSet命令集合系统、命令解析流水线
└── EventScheduler事件调度、GameEngine整合

阶段二：武侠世界构建（第4-5周）
├── Character属性系统（先天/后天/动态三层）
├── 装备系统（12+槽位、耐久度）
├── 武学系统（招式脚本沙箱、克制关系）
└── 地图系统（三维坐标、A*寻路）

阶段三：玩法系统实现（第6-7周）
├── 战斗系统（回合/即时双模式、BUFF/DEBUFF）
├── 任务系统（目标追踪、因果点）
├── NPC系统（行为树、好感度）
└── 自然语言命令解析

阶段四：GUI客户端开发（第8周）
├── PySide6集成（qasync桥接）
├── 核心面板（主视窗、状态、罗盘、背包）
└── 主题系统（QSS样式）

阶段五：内容制作与集成（第9-15周）
├── 世界构建（金庸经典场景：襄阳、洛阳、少林等）
├── 武学数据（十大门派基础武学）
├── 任务与剧情（30+主线、50+支线任务）
└── NPC数据（50+有完整对话的NPC）

阶段六：存档与系统功能（第16-17周）
├── 存档系统（MessagePack序列化、自动存档）
├── 后台管理系统（F12开发者模式）
└── 平衡性测试工具（战斗模拟器）
```

## 工作流程

### 顺序激活模式

mud-architect 采用**顺序激活**模式协调开发流程，每次只激活一个专家，完成后切换到下一个：

```
┌─────────────────────────────────────────────────────────┐
│  阶段 N                                                 │
│  ┌──────────────┐    ┌──────────────────┐              │
│  │ @mud-coder   │───→│ @mud-reviewer    │              │
│  │   (激活)     │    │    (激活)        │              │
│  └──────────────┘    └──────────────────┘              │
│         ↑                       │                       │
│         └───────────────────────┘ (不通过则循环)       │
│                                 ↓ (通过)               │
│                         [标记完成]                     │
└─────────────────────────────────────────────────────────┘
                           ↓
                    进入阶段 N+1
```

### 具体步骤

1. **启动阶段** - 查阅对应 phase 文档，了解阶段目标和模块清单
2. **生成激活指令** - 使用 templates.md 中的模板，输出激活 mud-coder 的指令
3. **用户执行** - 用户复制指令，在新对话中 `@mud-coder` 激活
4. **代码审查** - 代码完成后，输出激活 mud-reviewer 的指令
5. **用户执行** - 用户 `@mud-reviewer` 审查代码
6. **循环或推进** - 审查通过则标记完成，进入下一模块；不通过则回到步骤2修复
7. **阶段验收** - 所有模块完成后，对照验收标准检查，确认后进入下一阶段

**注意**：mud-architect 是唯一的持久化协调者，mud-coder 和 mud-reviewer 是单次激活的专家，完成后立即释放上下文。
