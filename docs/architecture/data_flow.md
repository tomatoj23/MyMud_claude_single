## 玩家输入处理

### 流程图

```
玩家输入
    │
    ▼
┌─────────────┐
│ GUI Input   │
│ 获取输入文本  │
└──────┬──────┘
       │
       ▼
┌─────────────┐
│ MessageBus  │
│ emit_text() │
└──────┬──────┘
       │
       ▼
┌─────────────┐
│GameEngine   │
│process_input│
└──────┬──────┘
       │
       ▼
┌─────────────┐
│CommandHandler
│   handle()  │
└──────┬──────┘
       │
       ├──▶ 解析命令 ──▶ 匹配CmdSet
       │
       ▼
┌─────────────┐
│ Command.run │
└──────┬──────┘
       │
       ▼
┌─────────────┐
│ Command.func│
│  执行逻辑   │
└──────┬──────┘
       │
       ├──▶ 修改游戏状态
       │
       ▼
┌─────────────┐
│   返回结果   │
│ MessageBus  │
└──────┬──────┘
       │
       ▼
┌─────────────┐
│ GUI 更新    │
│ 显示结果    │
└─────────────┘
```

### 详细说明

1. **输入捕获**: GUI层通过输入框捕获玩家输入
2. **命令解析**: `CommandHandler` 解析命令文本，分离命令和参数
3. **命令匹配**: 在 `CmdSet` 中匹配命令，支持前缀匹配
4. **权限检查**: 检查调用者是否有权限执行命令
5. **执行命令**: 调用 `Command.func()` 执行实际逻辑
6. **结果返回**: 通过 `MessageBus` 将结果发送到GUI

## 对象创建流程

### 流程图

```
调用创建
    │
    ▼
┌─────────────────────┐
│ ObjectManager.create │
│                     │
│ 1. 验证typeclass路径 │
│ 2. 准备初始数据      │
└──────────┬──────────┘
           │
           ▼
┌─────────────────────┐
│ DatabaseManager     │
│ 创建数据库记录        │
│                     │
│ INSERT INTO objects │
│ (key, typeclass,    │
│  attributes)        │
└──────────┬──────────┘
           │
           ▼
┌─────────────────────┐
│ 获取新对象ID         │
└──────────┬──────────┘
           │
           ▼
┌─────────────────────┐
│ TypeclassLoader     │
│ 加载类型类           │
└──────────┬──────────┘
           │
           ▼
┌─────────────────────┐
│ 实例化Typeclass      │
│ __init__()          │
└──────────┬──────────┘
           │
           ▼
┌─────────────────────┐
│ at_init()           │
│ 生命周期钩子         │
└──────────┬──────────┘
           │
           ▼
┌─────────────────────┐
│ L1 Cache            │
│ 缓存新对象           │
└──────────┬──────────┘
           │
           ▼
      返回对象实例
```

### 详细说明

1. **路径验证**: 验证 `typeclass_path` 是否有效
2. **数据库插入**: 创建数据库记录，获取自增ID
3. **类加载**: `TypeclassLoader` 动态加载类型类
4. **实例化**: 创建类型类实例，初始化 `AttributeHandler`
5. **生命周期**: 调用 `at_init()` 钩子
6. **缓存**: 将对象加入L1缓存

### 流程图

```
开始战斗
    │
    ▼
┌─────────────────┐
│ CombatSession() │

│   战斗循环       │◀────│   战斗结束？     │
│ _combat_loop()  │     │ (胜负/逃跑/超时) │
└────────┬────────┘     └─────────────────┘
         │                          │
         ▼                          │
┌─────────────────┐                 │
│  处理AI回合      │                 │
│ _process_ai()   │                 │
└────────┬────────┘                 │
         │                          │
         ▼                          │
┌─────────────────┐                 │
│   AI决策         │                 │
│ CombatAI.decide │                 │
└────────┬────────┘                 │
         │                          │
         ▼                          │
┌─────────────────┐                 │
│  执行行动        │                 │
│ _execute_action │                 │
└────────┬────────┘                 │
         │                          │
         ├──▶ 招式攻击              │
         │    _execute_move()       │
         │                          │
         ├──▶ 使用物品              │
         │                          │
         ├──▶ 逃跑                 │
         │                          │
         └──▶ 防御                 │
                                  │
         ▲                        │
         │                        │
         ▼                        ▼
┌─────────────────┐     ┌─────────────────┐
│  检查战斗结束    │────▶│     stop()      │
│  _check_end()   │     │  结算战斗结果    │
```

### 详细说明

1. **战斗初始化**: 创建 `CombatSession`，添加玩家和敌人
2. **AI决策**: `CombatAI` 根据当前状态决定行动
3. **行动执行**: 根据行动类型执行不同逻辑
4. **伤害计算**: `CombatCalculator` 计算命中率和伤害
5. **BUFF处理**: `BuffManager` 处理BUFF效果
6. **结束判定**: 检查胜负条件或逃跑成功

## 消息广播流程

### 流程图

```
消息源
    │
    ├──▶ 引擎事件
    │    (战斗/移动/获得物品)
    │
    ├──▶ 命令结果
    │    (查看/使用物品)
    │
    └──▶ 系统消息
         (保存/加载/错误)
         │
         ▼
┌─────────────────┐
│   MessageBus    │
│    emit()       │
│                 │
│ 1. 创建Message  │
│ 2. 遍历订阅者    │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│   按类型分发     │
│                 │
├──▶ Text消息 ────┼──▶ ScenePanel
│                 │
├──▶ System消息 ──┼──▶ LogPanel
│                 │
├──▶ Combat消息 ──┼──▶ CombatPanel
│                 │
└──▶ Dialogue消息 ▶──▶ DialoguePanel
```

### 消息类型

| 类型 | 用途 | 目标面板 |
|:---|:---|:---|
| `TEXT` | 普通文本输出 | ScenePanel, LogPanel |
| `SYSTEM` | 系统消息 | LogPanel |
| `COMBAT` | 战斗信息 | CombatPanel |
| `DIALOGUE` | 对话内容 | DialoguePanel |
| `ERROR` | 错误消息 | LogPanel |

## 自动保存流程

### 流程图

```
定时触发/手动触发
         │
         ▼
┌─────────────────┐
│  save_all()     │
│  批量保存       │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 获取脏数据列表   │
│ _dirty_objects  │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  批量序列化      │
│ to_db()         │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  数据库批量更新  │
│ UPDATE objects  │
│ SET attributes  │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  清除脏标记      │
│ _is_dirty=False │
└─────────────────┘
```

### 详细说明

1. **触发条件**: 定时器（默认5分钟）或重要事件
2. **脏数据收集**: 遍历缓存，找出标记为脏的对象
3. **批量序列化**: 将对象属性序列化为JSON
4. **批量更新**: 使用 `executemany` 批量更新数据库
5. **清除标记**: 更新成功后清除脏标记

## 数据持久化格式

### 对象表结构

```sql
CREATE TABLE objects (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    key TEXT NOT NULL,
    typeclass_path TEXT NOT NULL,
    location_id INTEGER REFERENCES objects(id),
    attributes JSON,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### 属性JSON格式

```json
{
    "level": 10,
    "exp": 1500,
    "status": {
        "hp": [85, 100],
        "mp": [45, 50]
    },
    "birth_talents": {
        "gengu": 20,
        "wuxing": 18,
        "fuyuan": 15,
        "rongmao": 16
    }
}
```

## 缓存一致性

### 缓存更新策略

| 操作 | 缓存行为 | 数据库行为 |
|:---|:---|:---|
| **读取** | 先查缓存，未命中则查库并缓存 | 查询 |
| **创建** | 加入缓存 | 插入 |
| **更新** | 更新缓存，标记脏数据 | 延迟批量更新 |
| **删除** | 从缓存移除 | 删除 |

### 缓存失效

- **过期策略**: 定期清理长时间未访问的对象
- **手动刷新**: 提供 `refresh()` 方法强制刷新
- **写穿透**: 重要操作立即写入数据库
