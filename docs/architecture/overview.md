## 分层架构

### 1. 表现层 (GUI Layer)

**职责**: 用户交互界面

**技术栈**:
- PySide6 (Qt6 Python绑定)
- qasync (asyncio与Qt桥接)
- QSS (样式表)

**组件**:
- `MainWindow`: 主窗口框架
- `ScenePanel`: 场景描述显示
- `CharacterPanel`: 角色属性面板
- `CombatPanel`: 战斗界面
- `DialoguePanel`: 对话界面
- `LogPanel`: 日志输出

**通信方式**:
- 通过 `MessageBus` 与引擎层通信
- 使用信号槽机制进行UI更新

---

### 2. 引擎层 (Engine Layer)

**职责**: 游戏核心逻辑

#### 2.1 核心系统

| 系统 | 职责 | 关键类 |
|:---|:---|:---|
| **GameEngine** | 引擎主控、生命周期管理 | `GameEngine` |
| **Typeclass** | 动态类型系统 | `TypeclassBase`, `TypeclassMeta` |
| **AttributeHandler** | 属性代理管理 | `AttributeHandler` |
| **MessageBus** | 消息分发 | `MessageBus`, `MessageHandler` |

#### 2.2 管理器系统

| 管理器 | 职责 | 关键类 |
|:---|:---|:---|
| **ObjectManager** | 对象CRUD、L1缓存 | `ObjectManager` |
| **CommandHandler** | 命令解析执行 | `CommandHandler`, `CmdSet` |
| **EventScheduler** | 延迟/周期事件 | `EventScheduler` |
| **DatabaseManager** | 数据库连接 | `DatabaseManager` |

---

### 3. 游戏层 (Game Layer)

**职责**: 武侠游戏具体实现

#### 3.1 类型类系统

```
TypeclassBase
    ├── Character (角色)
    │   ├── NPC (NPC)
    │   └── PlayerCharacter (玩家角色)
    ├── Room (房间)
    ├── Exit (出口)
    ├── Item (物品)
    │   └── Equipment (装备)
    └── Object (普通对象)
```

#### 3.2 游戏系统

| 系统 | 职责 | 关键模块 |
|:---|:---|:---|
| **战斗系统** | 回合制战斗、AI、伤害计算 | `combat/` |
| **武学系统** | 武功、招式、克制关系 | `wuxue/` |
| **NPC系统** | AI行为树、对话、好感度 | `npc/` |
| **任务系统** | 任务链、目标追踪 | `quest/` |
| **装备系统** | 装备槽、品质、耐久度 | `equipment/` |

## 关键设计模式

### 1. 类型类模式 (Typeclass Pattern)

**目的**: 实现动态类加载和属性代理

**实现**:
```python
class TypeclassBase(metaclass=TypeclassMeta):
    """通过元类实现动态注册。"""
    typeclass_path = "path.to.Class"
    
    def __init__(self, manager, db_model):
        self.db = AttributeHandler(db_model.attributes)
```

**优点**:
- 运行时动态加载类
- 属性变更自动追踪
- 支持继承和多态

---

### 2. 缓存模式 (Cache Pattern)

**L1缓存架构**:

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   Client    │────▶│  L1 Cache   │────▶│  Database   │
│             │◀────│   (Dict)    │◀────│             │
└─────────────┘     └─────────────┘     └─────────────┘
                           │
                    ┌──────┴──────┐
                    ▼             ▼
              ┌─────────┐   ┌─────────┐
              │  Hit    │   │  Miss   │
              │(直接返回)│   │(查库缓存)│
              └─────────┘   └─────────┘
```

**缓存策略**:
- 读: 先查缓存，未命中则查库并缓存
- 写: 更新对象并标记脏数据
- 定期批量保存脏数据

---

### 3. 命令模式 (Command Pattern)

**命令处理流程**:

```
输入文本
    │
    ▼
┌─────────────┐
│ CommandHandler.parse_input()
│   解析命令和参数
└──────┬──────┘
       │
       ▼
┌─────────────┐
│ CmdSet.match()
│   匹配命令
└──────┬──────┘
       │
       ▼
┌─────────────┐
│ Command.run()
│   执行命令
└──────┬──────┘
       │
       ▼
  返回结果
```

---

### 4. 观察者模式 (Observer Pattern)

**消息总线实现**:

```
┌───────────┐     ┌───────────┐     ┌───────────┐
│  Publisher  │────▶│ MessageBus │◀────│ Subscriber │
│  (引擎)     │     │  (中继站)   │     │  (GUI)    │
└───────────┘     └─────┬─────┘     └───────────┘
                        │
           ┌────────────┼────────────┐
           ▼            ▼            ▼
       ┌───────┐   ┌───────┐   ┌───────┐
       │Text   │   │System │   │Combat │
       │Message│   │Message│   │Message│
       └───────┘   └───────┘   └───────┘
```

## 扩展点

### 1. 添加新类型类

继承 `TypeclassBase`:

```python
class MyTypeclass(TypeclassBase):
    typeclass_path = "path.to.MyTypeclass"
    
    def at_init(self):
        # 自定义初始化
        pass
```

### 2. 添加新命令

继承 `Command`:

```python
class MyCommand(Command):
    key = "mycommand"
    
    async def func(self):
        # 命令逻辑
        pass
```

### 3. 添加新面板

继承 `QWidget` 并实现 `MessageHandler`:

```python
class MyPanel(QWidget, MessageHandler):
    def on_message(self, msg: Message):
        # 处理消息
        pass
```

## 性能考虑

### 优化策略

| 策略 | 实现 |
|:---|:---|
| **缓存** | L1缓存 + 数据库查询缓存 |
| **批量操作** | 脏数据批量保存 |
| **异步IO** | 所有数据库操作异步 |
| **懒加载** | 对象按需加载 |

### 性能指标

- 命令响应: < 100ms
- 内存占用: < 500MB
- 启动时间: < 5s
- 单场景对象: 100+
