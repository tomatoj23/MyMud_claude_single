## 🔍 架构层面长期债务

### 模块耦合问题

**分析**:
```
┌─────────────┐     ┌─────────────┐
│   combat    │◀───▶│   npc       │
│             │     │             │
│  直接依赖    │     │  直接依赖    │
│  Character  │     │  Character  │
└─────────────┘     └─────────────┘
         │                │
         └───────┬────────┘
                 ▼
          ┌─────────────┐
          │  Character  │
          │  职责过多    │
          └─────────────┘
```

**问题**: `Character`类职责过多（角色+装备+武学+属性）

**建议**: 考虑ECS架构或更细粒度的Mixin拆分

### 数据库设计限制

**问题**: 属性存储为JSON，查询能力受限

**建议**: 对常用字段建立JSON索引或规范化
